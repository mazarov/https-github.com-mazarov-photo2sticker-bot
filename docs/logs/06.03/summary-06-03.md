# Отчет по логам API/Worker (06.03)

Источник: логи из `docs/logs/06.03` за последние дни.

## 1) Паттерны поведения пользователей

- Основной путь: `start -> загрузка фото -> style generation -> пост-действия (emotion/motion/text)` и/или переход в `pack`.
- Часто встречаются ретраи через `retry_generation` после ошибки.
- Часть пользователей многократно отправляют `/start` в рамках одной сессии (переинициализация flow).
- Есть сегмент активных pack-пользователей: несколько успешных паков подряд.
- У части пользователей длинные цепочки `emotion/motion` (итеративная доработка одного стикера).

## 2) Проблемы, с которыми столкнулись

- `Gemini returned no image` (single-flow style/emotion/motion), даже после retry/fallback.
- `Gemini API failed: User location is not supported for the API use.`
- Многократные ошибки pack preview: `Не удалось сгенерировать превью` (с возвратом кредита).
- Ранее в assistant ideas наблюдались `stale_callback` (часть проблем уже исправлялась).

## 3) Загружали ли фото

Да. В логах есть множественные события `Photo received, telegramId: ...` и `Photo handler - session ...`.

## Резюме

- Платящие пользователи активны, но есть системные потери на `no-image` и `geo restriction`.
- Для роста конверсии важны: стабильность single-flow генерации, прозрачные причины ошибок, быстрый fallback/обход по региону.
- Для retention у активных pack-пользователей важнее UX-ускорение и предсказуемость preview.
