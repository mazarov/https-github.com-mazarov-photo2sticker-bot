# Изменения лендинга по фреймворку Pain → Hope → Solution

**Дата:** 17.02.2026  
**Основа:** [17-02-pain-hope-solution-framework.md](./17-02-pain-hope-solution-framework.md)  
**Цель:** Детально описать, что меняется на лендинге (субмодуль `landing/`) при внедрении воронки.

---

## 1. Структура страницы: было → стало

### Текущий порядок секций (`home.tsx`)

```
Hero → SocialProof → Reviews → StyleGallery → HowItWorks → Features → FAQ
+ Sticky CTA (внизу)
```

### Целевой порядок (по фреймворку)

```
Hero (хук + CTA)
→ Pain (новая секция)
→ More Pain (новая секция)
→ Hope (новая секция)
→ SocialProof
→ Reviews
→ StyleGallery
→ HowItWorks (решение: шаги)
→ Features (решение: фичи)
→ Блок цены (новый или внутри Features)
→ FAQ
+ Sticky CTA
```

**Правило:** После Hero идут три текстовых блока воронки (Боль → Усиление боли → Надежда), затем социальные доказательства и продукт (шаги, фичи, цена, FAQ).

## 2. Изменения по блокам

### 2.1. Hero

**Файл:** `landing/client/src/components/landing/Hero.tsx`

**Что меняется:** Тексты H1 и подзаголовка (дефолт + UTM-варианты). Продукт — **пак стикеров из фото**.

| Место | Сейчас | Нужно |
|-------|--------|--------|
| `DEFAULT_H1` | «Превращаем фото в стикеры для Telegram» | «Превращаем фото в пак стикеров для Telegram» |
| `DEFAULT_SUB` | «ИИ сохраняет черты лица и добавляет стиль. 30 секунд — и стикер готов» | «ИИ сохраняет черты лица. Выбери стиль и эмоции — 30 секунд, и готовый пак стикеров у тебя. Первый пак бесплатно.» |
| UTM `free`, `online` | h1: «Бесплатный стикер из фото — за 30 секунд», sub: «Без регистрации, без оплаты. Просто отправь фото боту» | h1: «Бесплатный пак стикеров из фото — за 30 секунд», sub: оставить или «Без регистрации, без оплаты. Просто пришли фото боту» |
| UTM `bot` | h1: «Telegram бот: фото → стикер за 30 секунд», sub: «Выбери стиль, добавь эмоции. Первый стикер бесплатно» | h1: «Telegram бот: фото → пак стикеров за 30 секунд», sub: «Выбери стиль и эмоции. Первый пак бесплатно» |
| UTM `ai`, `neural` | h1: «ИИ превращает фото в стикеры», sub: «Нейросеть сохраняет черты лица. 10+ стилей на выбор» | h1: «ИИ превращает фото в пак стикеров», sub: «Нейросеть сохраняет черты лица. Выбери стиль и эмоции — готовый пак за 30 сек» |
| UTM `telegram`, `tg` | h1: «Стикеры для Telegram из твоих фото», sub: «Отправь фото боту — стикер с твоим лицом за 30 сек» | h1: «Пак стикеров для Telegram из твоих фото», sub: «Пришли фото боту — пак стикеров с разными эмоциями за 30 сек» |

**Итог:** В `Hero.tsx` заменить константы `UTM_VARIANTS` и `DEFAULT_H1` / `DEFAULT_SUB` на тексты из таблицы выше.

### 2.2. Новая секция: Боль (Pain)

**Файл:** новый компонент `landing/client/src/components/landing/PainBlock.tsx` (или `PainHopeSection.tsx` с тремя подблоками — см. ниже).

**Назначение:** Показать, что мы понимаем боль пользователя. О продукте не говорим.

**Содержимое:**

| Элемент | Тип | Текст |
|---------|-----|--------|
| Заголовок | `h2` | Знакомо? |
| Текст | параграф | Хочется свой пак стикеров из фото для Telegram — с собой, с разными эмоциями, в любимом стиле. Но непонятно, как сделать без дизайнера и без часов в редакторе. Искать инструкции — муторно, приложения то платные, то с подпиской. В итоге пак стикеров так и остаётся «когда-нибудь». |

**Верстка:** Секция с тем же отступом/стилем, что остальные (`py-8 md:py-14 px-4 md:px-8`), заголовок по центру, текст под ним, можно в карточке или просто блок текста. Стили — в духе текущего лендинга (фон `bg-background`, текст `text-white` / `text-muted-foreground`).

**Подключение:** В `home.tsx` после `<Hero />` вставить `<PainBlock />` (или первый подблок общей секции).

### 2.4. Новая секция: Надежда (Hope)

## 3. Новые компоненты: структура

Рекомендуемая разметка для трёх новых блоков (Боль, Усиление боли, Надежда):

- Общая секция с `section`, классы как у других (например `py-8 md:py-14 px-4 md:px-8`).
- Заголовок: `h2` с классами как у других заголовков лендинга (например `text-lg sm:text-2xl md:text-3xl font-display font-bold text-white text-center mb-4`).
- Текст: один или два параграфа, `text-muted-foreground`, `text-center`, `max-w-2xl mx-auto` для читаемости.

**Вариант реализации:**

- **Вариант A:** Три отдельных компонента `PainBlock.tsx`, `MorePainBlock.tsx`, `HopeBlock.tsx` — каждый со своим заголовком и текстом (тексты захардкожены или через константы).
- **Вариант B:** Один компонент `PainHopeSection.tsx`, принимающий `variant: "pain" | "morePain" | "hope"` и рендерящий нужный заголовок и текст (контент в одном месте).

Для A/B-тестов и смены формулировок удобнее вынести тексты в константы или конфиг (например `landing/client/src/content/painHope.ts`).
